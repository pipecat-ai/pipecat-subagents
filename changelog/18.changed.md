- Bumped the minimum `pipecat-ai` requirement to `>=1.2.0`. The UI Agent Protocol is now a first-class part of the RTVI protocol: the wire format (envelope-type strings, paired `*Data` / `*Message` pydantic models, task-lifecycle `kind` discriminators, and the built-in command payload models `Toast`, `Navigate`, `ScrollTo`, `Highlight`, `Focus`, `Click`, `SetInputValue`, `SelectText`) lives in `pipecat.processors.frameworks.rtvi.models`. Type strings are kebab-case (`ui-event`, `ui-command`, `ui-snapshot`, `ui-cancel-task`, `ui-task`) to match the rest of the RTVI protocol. The bridge installed by `attach_ui_bridge` now subscribes to `on_ui_message` on the RTVI processor and emits typed envelopes by pushing the new pipecat `RTVIUICommandFrame` and `RTVIUITaskFrame` frames; the observer wraps each frame into the matching `UICommandMessage` / `UITaskMessage` envelope. Single-LLM Pipecat apps and other frameworks can target the same wire format directly without taking a subagents dependency. Removes the previous re-exports from `pipecat_subagents.agents` and `pipecat_subagents.bus`; import the wire-format names from `pipecat.processors.frameworks.rtvi.models` directly.
