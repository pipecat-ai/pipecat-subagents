- Bumped the minimum `pipecat-ai` requirement to `>=1.2.0`. The UI Agent Protocol wire format (envelope-type strings, reserved event names, task-lifecycle `kind` discriminators, and the built-in command payload models `Toast`, `Navigate`, `ScrollTo`, `Highlight`, `Focus`, `Click`, `SetInputValue`, `SelectText`) now lives in `pipecat.processors.frameworks.rtvi.models`. The payload models are pydantic `BaseModel`s instead of dataclasses; `UIAgent.send_command(...)` accepts both. Single-LLM Pipecat apps and other frameworks can now target the same wire format directly without taking a subagents dependency. Removes the previous re-exports from `pipecat_subagents.agents` and `pipecat_subagents.bus`; import the wire-format names from `pipecat.processors.frameworks.rtvi.models` directly.
